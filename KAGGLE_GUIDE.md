# Kaggle Guide: GPU Training With Small Data First

This guide is aligned with the current pipeline behavior:
- Position-0 tokens are dropped during activation collection.
- `run_sae.py` and `run_FC_sae.py` now expect a versioned activation artifact when using `--skip-collection`.

The workflow below starts with a tiny run (`~100` texts) so you can confirm everything works on Kaggle GPU before scaling.

## 4. Smoke Test: Standard SAE With ~100 Texts

This command does collection + training end-to-end using a tiny dataset.

```python
# Cell 2: tiny end-to-end SAE run (~100 texts)
!python run_sae.py \
    --layer 8 \
    --dataset openwebtext \
    --num-texts 100 \
    --samples 400 \
    --max-length 64 \
    --collection-batch-size 8 \
    --epochs 2 \
    --batch-size 64 \
    --expansion 4 \
    --l1-coeff 3e-4 \
    --lr 1e-3 \
    --disable-early-stopping \
    --save-every 1 \
    --log-every 1 \
    --checkpoint-dir checkpoints_sae_tiny \
    --activations-path activations_v2.pt \
    --device cuda
```

Expected outputs:
1. `activations_v2.pt` (new schema artifact)
2. `checkpoints_sae_tiny/best_model.pt`
3. `checkpoints_sae_tiny/training_history.png`
4. `checkpoints_sae_tiny/analysis_report.txt`

## 8. Important Compatibility Notes

1. If you pass `--skip-collection`, the file in `--activations-path` must be created by updated `run_sae.py` or `run_FC_sae.py`.
2. Legacy tensor-only activation files are intentionally rejected.
3. `collect_activation.py` currently saves a raw tensor file, so do not use that output with `--skip-collection` in the new training scripts.

## 9. Troubleshooting

1. CUDA OOM:
     1. Lower `--collection-batch-size` first.
     2. Lower `--batch-size` for training.
     3. Lower `--max-length`.
2. Slow run:
     1. Keep `--num-texts 100` and `--samples 400` for smoke tests.
     2. Use `--epochs 1` or `2` initially.
3. Dataset/model download issue:
     1. Ensure Kaggle internet is enabled.
     2. Retry once to populate cache.
